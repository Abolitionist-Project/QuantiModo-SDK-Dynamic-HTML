This is the API

userVariableRelationshipsGet

Get all UserVariableRelationships

URL

https://app.quantimo.do/api/v2/userVariableRelationships

HTTP Method

GET

Response Type

inline_response_200_8

Parameters

  • Query: access_token String

    User's OAuth2 access token

  • Query: id Integer

    id

  • Query: confidence_level String

    Our confidence that a consistent predictive relationship exists based on the amount of evidence, reproducibility, and other factors

  • Query: confidence_score BigDecimal

    A quantitative representation of our confidence that a consistent predictive relationship exists based on the amount of evidence, reproducibility, and other factors

  • Query: direction String

    Direction is positive if higher predictor values generally precede higher outcome values. Direction is negative if higher predictor values generally precede lower outcome values.

  • Query: duration_of_action Integer

    Estimated number of seconds following the onset delay in which a stimulus produces a perceivable effect

  • Query: error_message String

    error_message

  • Query: onset_delay Integer

    Estimated number of seconds that pass before a stimulus produces a perceivable effect

  • Query: outcome_variable_id Integer

    Variable ID for the outcome variable

  • Query: predictor_variable_id Integer

    Variable ID for the predictor variable

  • Query: predictor_unit_id Integer

    ID for default unit of the predictor variable

  • Query: sinn_rank BigDecimal

    A value representative of the relevance of this predictor relative to other predictors of this outcome. Usually used for relevancy sorting.

  • Query: strength_level String

    Can be weak, medium, or strong based on the size of the effect which the predictor appears to have on the outcome relative to other variable relationship strength scores.

  • Query: strength_score BigDecimal

    A value represented to the size of the effect which the predictor appears to have on the outcome.

  • Query: user_id Integer

    user_id

  • Query: vote String

    vote

  • Query: value_predicting_high_outcome BigDecimal

    Value for the predictor variable (in it's default unit) which typically precedes an above average outcome value

  • Query: value_predicting_low_outcome BigDecimal

    Value for the predictor variable (in it's default unit) which typically precedes a below average outcome value

  • Query: limit Integer

    The LIMIT is used to limit the number of results returned. So if you have 1000 results, but only want to the first 10, you would set this to 10 and offset to 0. The maximum limit is 200 records.

  • Query: offset Integer

    OFFSET says to skip that many rows before beginning to return rows to the client. OFFSET 0 is the same as omitting the OFFSET clause. If both OFFSET and LIMIT appear, then OFFSET rows are skipped before starting to count the LIMIT rows that are returned.

  • Query: sort String

    Sort by given field. If the field is prefixed with '-', it will sort in descending order.
