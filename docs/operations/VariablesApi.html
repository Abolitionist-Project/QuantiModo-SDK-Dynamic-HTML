publicVariablesGet

This endpoint retrieves an array of all public variables. Public variables are things like foods, medications, symptoms, conditions, and anything not unique to a particular user. For instance, a telephone number or name would not be a public variable.

URL

https://localhost/api/public/variables

HTTP Method

GET

Response Type

Parameters

publicVariablesSearchSearchGet

Get top 5 PUBLIC variables with the most correlations containing the entered search characters. For example, search for 'mood' as an effect. Since 'Overall Mood' has a lot of correlations with other variables, it should be in the autocomplete list.

URL

https://localhost/api/public/variables/search/{search}

HTTP Method

GET

Response Type

Parameters

  • Path: search String

    Search query can be some fraction of a variable name.

  • Query: effectOrCause String

    Allows us to specify which column in the `correlations` table will be searched. Choices are effect or cause.

variableUserSettingsPost

Users can change things like the display name for a variable. They can also change the parameters used in analysis of that variable such as the expected duration of action for a variable to have an effect, the estimated delay before the onset of action. In order to filter out erroneous data, they are able to set the maximum and minimum reasonable daily values for a variable.

URL

https://localhost/api/variableUserSettings

HTTP Method

POST

Response Type

Parameters

  • Body: sharingData List

    Variable user settings data

variablesSearchSearchGet

Get variables containing the search characters for which the currently logged in user has measurements. Used to provide auto-complete function in variable search boxes.

URL

https://localhost/api/variables/search/{search}

HTTP Method

GET

Response Type

Parameters

  • Path: search String

    Search query which may be an entire variable name or a fragment of one.

  • Query: categoryName String

    Filter variables by category name. The variable categories include Activity, Causes of Illness, Cognitive Performance, Conditions, Environment, Foods, Location, Miscellaneous, Mood, Nutrition, Physical Activity, Physique, Sleep, Social Interactions, Symptoms, Treatments, Vital Signs, and Work.

  • Query: source String

    Specify a data source name to only return variables from a specific data source.

  • Query: limit Integer

    The LIMIT is used to limit the number of results returned. So if you have 1000 results, but only want to the first 10, you would set this to 10 and offset to 0.

  • Query: offset Integer

    Now suppose you wanted to show results 11-20. You'd set the offset to 10 and the limit to 10.
