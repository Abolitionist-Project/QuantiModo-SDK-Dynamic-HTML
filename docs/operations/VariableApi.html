This is the API

variablesGet

Get all Variables

URL

https://app.quantimo.do/api/v2/variables

HTTP Method

GET

Response Type

inline_response_200_34

Parameters

  • Query: access_token String

    User's OAuth2 access token

  • Query: id Integer

    id

  • Query: client_id String

    The ID of the client application which last created or updated this common variable

  • Query: parent_id Integer

    ID of the parent variable if this variable has any parent

  • Query: name String

    User-defined variable display name

  • Query: variable_category_id Integer

    Variable category ID

  • Query: default_unit_id Integer

    ID of the default unit for the variable

  • Query: combination_operation String

    How to combine values of this variable (for instance, to see a summary of the values over a month) SUM or MEAN

  • Query: filling_value BigDecimal

    Value for replacing null measurements

  • Query: maximum_allowed_value BigDecimal

    Maximum reasonable value for this variable (uses default unit)

  • Query: minimum_allowed_value BigDecimal

    Minimum reasonable value for this variable (uses default unit)

  • Query: onset_delay Integer

    Estimated number of seconds that pass before a stimulus produces a perceivable effect

  • Query: duration_of_action Integer

    Estimated number of seconds following the onset delay in which a stimulus produces a perceivable effect

  • Query: public Integer

    Is variable public

  • Query: cause_only Boolean

    A value of 1 indicates that this variable is generally a cause in a causal relationship. An example of a causeOnly variable would be a variable such as Cloud Cover which would generally not be influenced by the behaviour of the user

  • Query: most_common_value BigDecimal

    Most common value

  • Query: most_common_unit_id Integer

    Most common Unit

  • Query: standard_deviation BigDecimal

    Standard Deviation

  • Query: variance BigDecimal

    Average variance for this variable based on all user data

  • Query: mean BigDecimal

    Mean for this variable based on all user data

  • Query: median BigDecimal

    Median for this variable based on all user data

  • Query: number_of_measurements BigDecimal

    Number of measurements for this variable based on all user data

  • Query: number_of_unique_values BigDecimal

    Number of unique values for this variable based on all user data

  • Query: skewness BigDecimal

    Skewness for this variable based on all user data

  • Query: kurtosis BigDecimal

    Kurtosis for this variable based on all user data

  • Query: status String

    status

  • Query: error_message String

    error_message

  • Query: last_successful_update_time String

    When this variable or its settings were last updated

  • Query: created_at String

    When the record was first created. Use ISO 8601 datetime format

  • Query: updated_at String

    When the record was last updated. Use ISO 8601 datetime format

  • Query: product_url String

    Product URL

  • Query: image_url String

    Image URL

  • Query: price BigDecimal

    Price

  • Query: number_of_user_variables Integer

    Number of users who have data for this variable

  • Query: outcome Boolean

    Outcome variables (those with `outcome` == 1) are variables for which a human would generally want to identify the influencing factors. These include symptoms of illness, physique, mood, cognitive performance, etc. Generally correlation calculations are only performed on outcome variables.

  • Query: minimum_recorded_value BigDecimal

    Minimum recorded value of this variable based on all user data

  • Query: maximum_recorded_value BigDecimal

    Maximum recorded value of this variable based on all user data

  • Query: limit Integer

    The LIMIT is used to limit the number of results returned. So if you have 1000 results, but only want to the first 10, you would set this to 10 and offset to 0. The maximum limit is 200 records.

  • Query: offset Integer

    OFFSET says to skip that many rows before beginning to return rows to the client. OFFSET 0 is the same as omitting the OFFSET clause. If both OFFSET and LIMIT appear, then OFFSET rows are skipped before starting to count the LIMIT rows that are returned.

  • Query: sort String

    Sort records by a given field name. If the field name is prefixed with '-', it will sort in descending order.

  • Body: body Variable

    Variable that should be stored
