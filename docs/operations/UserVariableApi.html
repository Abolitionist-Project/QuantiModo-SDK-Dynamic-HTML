userVariablesGet

Get all UserVariables

URL

https://app.quantimo.do/api/v2/userVariables

HTTP Method

GET

Response Type

inline_response_200_9

Parameters

  • Query: access_token String

    User's OAuth2 access token

  • Query: client_id String

    The ID of the client application which last created or updated this user variable

  • Query: parent_id Integer

    ID of the parent variable if this variable has any parent

  • Query: variable_id Integer

    ID of variable

  • Query: user_id Integer

    User ID

  • Query: default_unit_id Integer

    D of unit to use for this variable

  • Query: minimum_allowed_value BigDecimal

    Minimum reasonable value for this variable (uses default unit)

  • Query: maximum_allowed_value BigDecimal

    Maximum reasonable value for this variable (uses default unit)

  • Query: filling_value BigDecimal

    Value for replacing null measurements

  • Query: join_with Integer

    The Variable this Variable should be joined with. If the variable is joined with some other variable then it is not shown to user in the list of variables

  • Query: onset_delay Integer

    Estimated number of seconds that pass before a stimulus produces a perceivable effect

  • Query: duration_of_action Integer

    Estimated duration of time following the onset delay in which a stimulus produces a perceivable effect

  • Query: variable_category_id Integer

    ID of variable category

  • Query: updated Integer

    updated

  • Query: public Integer

    Is variable public

  • Query: cause_only Boolean

    A value of 1 indicates that this variable is generally a cause in a causal relationship. An example of a causeOnly variable would be a variable such as Cloud Cover which would generally not be influenced by the behaviour of the user

  • Query: filling_type String

    0 -> No filling, 1 -> Use filling-value

  • Query: number_of_measurements Integer

    Number of measurements

  • Query: number_of_processed_measurements Integer

    Number of processed measurements

    Number of measurements at last analysis

    ID of last Unit

  • Query: last_original_unit_id Integer

    ID of last original Unit

  • Query: last_original_value Integer

    Last original value which is stored

    Last Value

  • Query: last_original_value BigDecimal

    Last original value which is stored

    ID of last source

    Number of correlations for this variable

  • Query: status String

    status

    error_message

  • Query: last_successful_update_time String

    When this variable or its settings were last updated

  • Query: standard_deviation BigDecimal

    Standard deviation

  • Query: variance BigDecimal

    Variance

    Minimum recorded value of this variable

  • Query: maximum_recorded_value BigDecimal

    Maximum recorded value of this variable

  • Query: mean BigDecimal

    Mean

  • Query: median BigDecimal

    Median

    Most common Unit ID

  • Query: most_common_value BigDecimal

    Most common value

  • Query: number_of_unique_daily_values BigDecimal

    Number of unique daily values

  • Query: number_of_changes Integer

    Number of changes

  • Query: skewness BigDecimal

    Skewness

    Kurtosis

    Latitude

  • Query: longitude BigDecimal

    Longitude

  • Query: location String

    Location

  • Query: created_at String

    When the record was first created. Use ISO 8601 datetime format

  • Query: updated_at String

    When the record was last updated. Use ISO 8601 datetime format

  • Query: outcome Boolean

    Outcome variables (those with `outcome` == 1) are variables for which a human would generally want to identify the influencing factors. These include symptoms of illness, physique, mood, cognitive performance, etc. Generally correlation calculations are only performed on outcome variables

  • Query: sources String

    Comma-separated list of source names to limit variables to those sources

  • Query: earliest_source_time Integer

    Earliest source time

  • Query: latest_source_time Integer

    Latest source time

  • Query: earliest_measurement_time Integer

    Earliest measurement time

  • Query: latest_measurement_time Integer

    Latest measurement time

  • Query: earliest_filling_time Integer

    Earliest filling time

  • Query: latest_filling_time Integer

    Latest filling time

  • Query: limit Integer

    The LIMIT is used to limit the number of results returned. So if you have 1000 results, but only want to the first 10, you would set this to 10 and offset to 0. The maximum limit is 200 records.

  • Query: offset Integer

    OFFSET says to skip that many rows before beginning to return rows to the client. OFFSET 0 is the same as omitting the OFFSET clause. If both OFFSET and LIMIT appear, then OFFSET rows are skipped before starting to count the LIMIT rows that are returned.

  • Query: sort String

    Sort by given field. If the field is prefixed with '-', it will sort in descending order.

userVariablesPost

Users can change things like the display name for a variable. They can also change the parameters used in analysis of that variable such as the expected duration of action for a variable to have an effect, the estimated delay before the onset of action. In order to filter out erroneous data, they are able to set the maximum and minimum reasonable daily values for a variable.

URL

https://app.quantimo.do/api/v2/userVariables

HTTP Method

POST

Response Type

inline_response_200_30

Parameters

  • Query: access_token String

    User's OAuth2 access token

  • Body: body UserVariable

    UserVariable that should be stored
