This is the API

trackingRemindersGet

Users can be reminded to track certain variables at a specified frequency with a default value.

URL

https://app.quantimo.do/api/v2/trackingReminders

HTTP Method

GET

Response Type

inline_response_200_6

Parameters

  • Query: access_token String

    User's OAuth2 access token

  • Query: client_id String

    The ID of the client application which last created or updated this tracking reminder

  • Query: user_id Integer

    ID of the user who created a reminder

  • Query: variable_id Integer

    Id for the variable to be tracked

  • Query: pop_up Boolean

    True if the reminders should appear as a popup notification

  • Query: sms Boolean

    True if the reminders should be delivered via SMS

  • Query: email Boolean

    True if the reminders should be delivered via email

  • Query: notification_bar Boolean

    True if the reminders should appear in the notification bar

  • Query: last_reminded String

    ISO 8601 timestamp for the last time a reminder was sent

  • Query: last_tracked String

    ISO 8601 timestamp for the last time a measurement was received for this user and variable

  • Query: created_at String

    When the record was first created. Use ISO 8601 datetime format

  • Query: updated_at String

    When the record was last updated. Use ISO 8601 datetime format

  • Query: limit Integer

    The LIMIT is used to limit the number of results returned. So if you have 1000 results, but only want to the first 10, you would set this to 10 and offset to 0. The maximum limit is 200 records.

  • Query: offset Integer

    OFFSET says to skip that many rows before beginning to return rows to the client. OFFSET 0 is the same as omitting the OFFSET clause. If both OFFSET and LIMIT appear, then OFFSET rows are skipped before starting to count the LIMIT rows that are returned.

  • Query: sort String

    Sort by given field. If the field is prefixed with '-', it will sort in descending order.

trackingRemindersPost

This is to enable users to indicate their opinion on the plausibility of a causal relationship between a treatment and outcome. QuantiModo incorporates crowd-sourced plausibility estimations into their algorithm. This is done allowing user to indicate their view of the plausibility of each relationship with thumbs up/down buttons placed next to each prediction.

URL

https://app.quantimo.do/api/v2/trackingReminders

HTTP Method

POST

Response Type

inline_response_200_23

Parameters

  • Query: access_token String

    User's OAuth2 access token

  • Body: body TrackingReminder

    TrackingReminder that should be stored
