v1ConnectJsGet

Get embeddable connect javascript. Usage: - Embedding in applications with popups for 3rd-party authentication windows. Use `qmSetupInPopup` function after connecting `connect.js`. - Embedding in applications with popups for 3rd-party authentication windows. Requires a selector to block. It will be embedded in this block. Use `qmSetupOnPage` function after connecting `connect.js`. - Embedding in mobile applications without popups for 3rd-party authentication. Use `qmSetupOnMobile` function after connecting `connect.js`. if using the MoodiModo Clones, Use `qmSetupOnIonic` function after connecting `connect.js`.

URL

https://app.quantimo.do/api/v1/connect.js

HTTP Method

GET

Response Type

Parameters

  • Query: access_token String

    User's OAuth2 access token

v1ConnectMobileGet

This page is designed to be opened in a webview. Instead of using popup authentication boxes, it uses redirection. You can include the user's access_token as a URL parameter like https://app.quantimo.do/api/v1/connect/mobile?access_token=123

URL

https://app.quantimo.do/api/v1/connect/mobile

HTTP Method

GET

Response Type

Parameters

  • Query: access_token String

    User OAuth access token

v1ConnectorsConnectorConnectGet

Attempt to obtain a token from the data provider, store it in the database. With this, the connector to continue to obtain new user data until the token is revoked.

URL

https://app.quantimo.do/api/v1/connectors/{connector}/connect

HTTP Method

GET

Response Type

Parameters

  • Path: connector String

    Lowercase system name of the source application or device. Get a list of available connectors from the /connectors/list endpoint.

  • Query: access_token String

    User's OAuth2 access token

v1ConnectorsConnectorConnectInstructionsGet

Returns instructions that describe what parameters and endpoint to use to connect to the given data provider.

URL

https://app.quantimo.do/api/v1/connectors/{connector}/connectInstructions

HTTP Method

GET

Response Type

Parameters

  • Path: connector String

    Lowercase system name of the source application or device. Get a list of available connectors from the /connectors/list endpoint.

  • Query: parameters String

    JSON Array of Parameters for the request to enable connector.

  • Query: url String

    URL which should be used to enable the connector.

  • Query: usePopup Boolean

    Should use popup when enabling connector

  • Query: access_token String

    User's OAuth2 access token

v1ConnectorsConnectorConnectParameterGet

Returns instructions that describe what parameters and endpoint to use to connect to the given data provider.

URL

https://app.quantimo.do/api/v1/connectors/{connector}/connectParameter

HTTP Method

GET

Response Type

ConnectorInstruction

Parameters

  • Path: connector String

    Lowercase system name of the source application or device. Get a list of available connectors from the /connectors/list endpoint.

  • Query: displayName String

    Name of the parameter that is user visible in the form

  • Query: key String

    Name of the property that the user has to enter such as username or password Connector (used in HTTP request)

  • Query: placeholder String

    Placeholder hint value for the parameter input tag.

  • Query: type String

    Type of input field such as those found here http://www.w3schools.com/tags/tag_input.asp

  • Query: usePopup Boolean

    Should use popup when enabling connector

  • Query: access_token String

    User's OAuth2 access token

  • Query: defaultValue String

    Default parameter value
